Simple Speed Control

The image below shows the abstraction of a simple speed control system:

On the left side, we have the reference speed input ωs, the set point. The feedback speed value ωf is subtracted from the set speed and fed as error signal δω into the speed controller WC(s). The controller implements the control algorithm, usually a PI or PID function.

Then follows the actuator WA(s), which converts the torque reference signal Tr into the torque output Tm. This can be a for example DC or AC motor, a stepper or a BLDC motor. The motor output torque Tm appears at the drive shaft.

The load torque input TL represents the load variations and disturbances. It counteracts the motor torque, therefore is subtracted from the motor torque. The sum is fed into the mechanical subsystem WP, which models load inertia and friction.

In a closed loop system, this block has an output ωa that represents the actual rotation speed. This signal is often generated by a digital rotary encoder or an AC generator. Some noise is added to the actual speed signal, the sum is fed into the speed acquisition block WE, which converts the actual rotation speed into the feedback speed ωf. This signal is then subtracted from the set speed ωs, as already described above.

A system as shown above is called a closed loop control system, because the output is fed back to and subtracted from the input, the set point. A system without feedback is called an open loop control system.

The controller in a closed loop system tries to minimize the speed error Δω. In other words, the controller should compensate the effect of friction Bω, inertial forces Jdωdt Jdω/dt, and load variations TL. When the system eventually settles to a steady state, the speed error Δω is ideally zero. Practically, this error should be smaller than a practical limit.

Transfer Functions

The transfer function of the mechanical subsystem is

WP(s)= 1 Js+B

where B represent the friction, and J represents the mechanical inertia.

In the time domain, this equation is

WP(t)= 1 Jdωdt+Bω

The following equation describes the mechanical forces:

Jdωdt =Tm−TL−Bω

The motor torque balances the load torque, friction and inertial force.

The ideal transfer function of the actuator (motor) is strictly linear without delays, so that the torque output TA accurately tracks the torque reference signal Tr :

WA(s)=kA

with kA=constant.

In practise, the inductivity of motor windings limit the current rise time. This can be compensated with a high voltage up to a certain limi. Therefore, a kA=constant is hardly achievable. However, the closed control loop cutoff-frequency is usually much (~100 times) higher than the expected response time of the mechanical system. Then the current rise time can safely be neglected and kA=constant holds.

The feedback signal ωf is derived from the actual speed ωa by the transfer function WF. Due to the limited resolution of the sensor, sampling and reconstruction, and the need to filter out noise and high-frequency interference, the feedback signal is not an exact copy of the actual speed. An RC filter for example implements the transfer function

WF(s)= 1 1+sRC

However, if the time constants (τ=RC) in the transfer function are considerably smaller than the desired speed response times, the transfer function can be considered constant:
